## 1. Аннотация программного продукта

Данный репозиторий содержит исходный код десктопного приложения, моделирующего работу игрового автомата типа «слот-машина». Программный продукт разработан на платформе .NET Framework с использованием объектно-ориентированного языка программирования C# и технологии построения графического интерфейса Windows Forms. 

Целью проекта является практическая реализация алгоритмов генерации псевдослучайных чисел, построение сбалансированной математической модели системы, а также применение паттернов проектирования для обеспечения жесткого разделения бизнес-логики и пользовательского интерфейса (Layered Architecture).

## 2. Теоретические основы и математическая модель

В основе функционирования системы лежит генератор псевдослучайных чисел (ГПЧ). Дискретное пространство элементарных исходов моделируется законами комбинаторики. 

Пусть автомат состоит из $R$ независимых барабанов, каждый из которых содержит $S$ уникальных символов. Общее число возможных исходов вращения ($N$) рассчитывается по формуле размещения с повторениями:

$$ N = S^R $$

В реализованной модели используются параметры $R = 3$ и $S = 5$. Следовательно, общее количество равновероятных комбинаций равно:

$$ N = 5^3 = 125 $$

Ключевым экономическим показателем системы является математическое ожидание игрока, выражаемое параметром RTP (Return to Player — теоретический возврат средств). Параметр вычисляется как сумма произведений вероятностей выпадения комбинаций на их соответствующие коэффициенты выплат:

$$ RTP = \sum_{i=1}^{n} (P_i \times M_i) \times 100\% $$

где:
* $P_i$ — вероятность наступления $i$-го выигрышного события;
* $M_i$ — множитель ставки для $i$-го события;
* $n$ — общее количество выигрышных комбинаций.

В таблице 1 приведено распределение вероятностей и коэффициентов для разработанной модели.

**Таблица 1 — Распределение вероятностей выигрышных комбинаций**

| Комбинация | Условие совпадения | Количество вариантов | Вероятность ($P_i$) | Коэффициент ($M_i$) |
| :--- | :--- | :--- | :--- | :--- |
| Джекпот | Три символа «Семерка» строго на всех барабанах | 1 | 0.008 (0.8%) | 10 |
| Три совпадения | Любые три одинаковых символа (исключая семерки) | 4 | 0.032 (3.2%) | 5 |
| Пара | Совпадение любых двух символов на любых позициях | 60 | 0.480 (48.0%) | 2 |

*Примечание: при указанных в таблице коэффициентах математическое ожидание превышает 100%, что требует дальнейшей корректировки коэффициента за комбинацию «Пара» для обеспечения рентабельности системы в условиях реальной эксплуатации.*

## 3. Архитектурные решения и структуры данных

Проект спроектирован с соблюдением принципов SOLID и разделен на две независимые сборки:
1. **Bandit.Logic** — слой предметной области. Содержит математический аппарат, систему обработки финансовых транзакций, алгоритмы ГПЧ и учет статистики. Классы логики инкапсулированы и не имеют зависимостей от графической подсистемы.
2. **Bandit.UI** — слой представления. Отвечает исключительно за рендеринг графических компонентов, обработку пользовательского ввода и таймеры анимации.

### Обоснование выбора типов данных
Для проведения финансовых операций (учет баланса, ставок, расчет выигрышей) в проекте использован 128-битный структурный тип данных `decimal`. Применение типов с плавающей запятой стандарта IEEE 754 (`double` или `float`) признано недопустимым ввиду возникновения погрешностей при операциях с десятичными дробями.

Для ведения истории транзакций применена динамическая типизированная структура `List<GameRecord>`. В целях оптимизации потребления оперативной памяти реализован алгоритм кольцевого буфера, жестко ограничивающий размер коллекции до 50 последних элементов, предотвращая утечки памяти при длительных сессиях.

## 4. Модульное тестирование

Верификация и валидация алгоритмов проведена с использованием встроенного фреймворка `MSTest`. Набор тестов спроектирован по паттерну AAA (Arrange — Act — Assert). 

Произведено покрытие следующих сценариев:
* Проверка целостности математических расчетов выигрыша (поведение `SlotMachine`).
* Валидация граничных условий при работе с балансом пользователя (перехват `ArgumentException` при попытке внесения отрицательного депозита и `InvalidOperationException` при недостаточности средств).
* Точность вычисления агрегированных статистических метрик (WinRate, NetProfit).
* Контроль переполнения памяти в модуле истории.

## 5. Инструкция по сборке и запуску

Для компиляции и запуска программного продукта необходима среда разработки, поддерживающая платформу .NET Framework (рекомендуется Microsoft Visual Studio 2019/2022).

**Порядок запуска:**
1. Клонировать репозиторий в локальную файловую систему.
2. Открыть файл решения `.sln` в IDE Visual Studio.
3. Установить проект `Bandit.UI` в качестве загружаемого проекта по умолчанию.
4. Выполнить сборку решения (Build Solution, клавиша `F6`).
5. Запустить приложение в режиме отладки (Start Debugging, клавиша `F5`).

Для запуска модульных тестов необходимо перейти в меню *Test -> Run All Tests* (или использовать комбинацию `Ctrl+R, A`). Тестовые сборки располагаются в проекте `Bandit.Tests`.
